# Sprint 1.2 — Excel 批量上传运费

> **Goal**: 团长收齐订单结算运费 / 重量 / 快递单号后,通过上传 xlsx 一键批量更新订单,不用一个个改。
>
> **Scope 边界**: 只做运费上传(更新 `order.shippingFee` + `order.tracking`)。**不含**:批量创建商品(Sprint 3)、批量发货状态(暂无)、用户管理(Sprint 3)。
>
> **配套**: `docs/backlog.md`(全 batch 总目录)、`docs/superpowers/specs/2026-05-18-sprint1.1-data-schema-oauth-cleanup-design.md`(前置 schema)。

---

## Context

Sprint 1.1 落地了:
- `order.shippingFee = {amount, outTradeNo, payStatus, setAt, paidAt}` — 单笔 `_admin/setShippingFee` cf
- `order.tracking = {weight, courierName, courierNo, setAt}` — 单笔 `_admin/updateTracking` cf
- admin OrderDetail 加了对应 UI(单笔操作)

实际生产中团长一次结算几十到上百个订单,单笔操作完全不现实。Sprint 1.2 加 Excel 批量入口,运营在 Excel 里填好统一上传,5 秒搞定 100 单。

OAuth-only 用户群体的下游(发货 / 通知)效率取决于这个。

## Section 1: 后端 cf `_admin/uploadShippingFeesXlsx`

```
入参: { xlsxBase64: string, dryRun: boolean }
出参: {
  code: 0,
  rows: BatchShippingRow[],
  summary: {
    matched: number,
    notFound: number,
    alreadyPaid: number,
    invalid: number,
    duplicateInFile: number,
    applied?: number,    // 只有 dryRun=false 时返
  },
}

type BatchShippingRow = {
  row: number,                          // xlsx 行号(2-based,header 是 1)
  orderNo: string,
  weight: number | null,                // kg
  fee: number | null,                   // cents(¥ × 100,Math.round)
  courierNo: string | null,
  status: 'matched' | 'not_found' | 'already_paid' | 'invalid' | 'duplicate_in_file',
  message?: string,                     // 错误说明(invalid 时填)
  before?: { shippingFee?: object, tracking?: object },  // 当前 order 已有数据(覆盖前)
}
```

### 处理流程

1. 鉴权:沿用 `_admin/*` requireAdmin pattern(token + admins collection)
2. 校验 `xlsxBase64`:存在且 base64 体积 < 2MB,否则 code 1 拒
3. base64 → Buffer → exceljs `Workbook.xlsx.load`,取第一个 sheet
4. 校验 header(第一行):必须含 4 列 `订单号 / 实际总重量 / 应补尾款 / 快递单号`(顺序可变,列名严格匹配)。任一缺失返 code 2。
5. 校验数据行数 ≤ 500,否则 code 3 拒
6. 遍历每行(从第 2 行开始):
   - 抽 4 字段
   - 单 row 校验:`订单号` 非空字符串、`重量` 数字 0..1000、`尾款` 数字 0..¥10000(0..1_000_000 分)、`单号` 字符串 ≤100 字符
   - 校验失败 → status `'invalid'` + message
   - 若 orderNo 已在本次上传 rows 数组中出现过 → status `'duplicate_in_file'`(第一次正常,后面标 duplicate;计入 summary.duplicateInFile)
   - 否则查 mongo orders by orderNo:
     - 不存在 → `'not_found'`
     - 存在 + `shippingFee.payStatus === 'paid'` → `'already_paid'`
     - 否则 → `'matched'`,并在 row.before 记当前 shippingFee/tracking
7. `dryRun === false` 时,对 `status === 'matched'` 行实际 update:
   ```js
   const now = new Date();
   const outTradeNo = 'SHIP' + Date.now() + crypto.randomBytes(4).toString('hex').toUpperCase();
   await orders.doc(order._id).update({
     data: {
       shippingFee: {
         amount: row.fee,
         outTradeNo,
         payStatus: 'pending',
         setAt: now,
         paidAt: null,
       },
       tracking: {
         weight: row.weight,
         courierName: null,          // Excel 没此列,留空,admin 后续单笔补
         courierNo: row.courierNo,
         setAt: now,
       },
       updatedAt: now,
     },
   });
   ```
8. 返 rows + summary。summary.applied 只在 dryRun=false 时填(applied = matched 数;skipped = total - matched)
9. console.log audit:`[uploadShippingFeesXlsx] admin=<openid> total=N applied=A skipped=S`(仅 commit 时)

### 关键设计点

- **No transaction**:per-row update,一行失败不影响其他
- **outTradeNo 每行新生成**:跟 setShippingFee 同款 hex 防重
- **courierName 留 null**:Excel 没此列,admin 后续在单笔 OrderDetail 手动补充
- **dryRun 幂等**:不写 mongo,可反复
- **upload 体积 / 行数限制**:防 cf 超时(2MB / 500 行)
- **重复 orderNo 处理**:第一次正常,后续行标 duplicate_in_file(不写 mongo,前端报告)

### API caller

`web-admin/src/api/order.ts` 加:

```ts
export interface BatchShippingRow {
  row: number;
  orderNo: string;
  weight: number | null;
  fee: number | null;
  courierNo: string | null;
  status: 'matched' | 'not_found' | 'already_paid' | 'invalid' | 'duplicate_in_file';
  message?: string;
  before?: {
    shippingFee?: { amount?: number; payStatus?: string };
    tracking?: { weight?: number; courierNo?: string };
  };
}

export interface BatchShippingResp {
  code: 0;
  rows: BatchShippingRow[];
  summary: {
    matched: number;
    notFound: number;
    alreadyPaid: number;
    invalid: number;
    duplicateInFile: number;
    applied?: number;
  };
}

export async function uploadShippingFeesXlsx(xlsxBase64: string, dryRun: boolean) {
  return callCloud<BatchShippingResp>('_admin/uploadShippingFeesXlsx', { xlsxBase64, dryRun });
}
```

### 文件读 → base64

浏览器原生 FileReader:

```ts
async function fileToBase64(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => {
      const result = reader.result as string;
      // data:application/...;base64,XXXX  → 切前缀
      const base64 = result.split(',')[1] || result;
      resolve(base64);
    };
    reader.onerror = reject;
    reader.readAsDataURL(file);
  });
}
```

### 模板下载 util

```ts
import ExcelJS from 'exceljs';

export async function downloadShippingTemplate() {
  const wb = new ExcelJS.Workbook();
  const ws = wb.addWorksheet('运费');
  ws.addRow(['订单号', '实际总重量', '应补尾款', '快递单号']);
  ws.addRow(['MG2026051820003932AA07', 2.5, 35.00, 'SF1234567890']);
  ws.getRow(1).font = { bold: true };
  ws.columns = [{ width: 28 }, { width: 12 }, { width: 12 }, { width: 22 }];
  const buf = await wb.xlsx.writeBuffer();
  const blob = new Blob([buf], {
    type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
  });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = '运费上传模板.xlsx';
  a.click();
  URL.revokeObjectURL(url);
}
```

## Section 3: 测试 + Rollback

### 测试(`local-backend/api/test-shim.js`)

测试需要生成 xlsx,加一个 helper:

```js
const ExcelJS = require('exceljs');

async function makeTestXlsx(rows) {
  // rows: [[订单号, 重量, 尾款, 单号], ...]
  const wb = new ExcelJS.Workbook();
  const ws = wb.addWorksheet('运费');
  ws.addRow(['订单号', '实际总重量', '应补尾款', '快递单号']);
  rows.forEach(r => ws.addRow(r));
  const buf = await wb.xlsx.writeBuffer();
  return Buffer.from(buf).toString('base64');
}
```

4 个新测试:

1. **dryRun 预览覆盖 5 种状态** — fixtures 准备订单状态各异,xlsx 包含 5 行,断言每行 status 对、summary 各字段正确、`summary.applied === undefined`
2. **apply 只写 matched 行** — 同 fixtures + dryRun=false,断言 mongo 上 matched 订单写入了 shippingFee+tracking,其他订单未变
3. **非 admin 拒** — 无 admin 用户调用,返 403
4. **header 错乱拒** — xlsx 只 3 列 / 列名拼错,返 code 2

预期 test 数 51 → 55。

### Rollback / 边界

- **No transaction**:per-row update,一行失败不影响其他
- **outTradeNo 唯一**:crypto.randomBytes(4) hex,撞概率 ~0
- **xlsx 上限**:行数 > 500 或 base64 > 2MB,cf 内拒 + 返 code 1/3
- **重复 dryRun**:不写 mongo,无副作用
- **commit 之后**:返结果显示 applied + skipped,前端关 modal 后 Orders reload
- **审计 log**:每次 apply 在 docker logs 留一行 `[uploadShippingFeesXlsx] admin=X total=N applied=A skipped=S`
- **真要回滚**:admin 在 OrderDetail 单笔重新设运费(把错的 shippingFee.payStatus='paid' 标记前可改)
