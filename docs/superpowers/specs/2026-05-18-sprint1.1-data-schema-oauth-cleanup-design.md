# Sprint 1.1 — 数据 schema 扩展 + OAuth-only 身份模型清理

> **Goal**: 落地甲方 Batch 1 中的 P0 schema 变化(群号/订单 tracking/卖家备注/商品扩展),同时完全清除遗留的 OTP 代码,把身份模型重构为 OAuth-only(`wechat + groupId` 作为身份,`address` 作为联系方式)。
>
> **Scope 边界**: 不含 Excel 批量上传运费(那是 Sprint 1.2 独立 spec)、不含 Sprint 2 / 3 任何项。
>
> **配套**: `docs/backlog.md`(全 batch 总目录)、`docs/decisions/2026-05-16-auth-and-notifications.md`(OAuth 决策)。

## Section 1: Data schema 变化

MongoDB schemaless,所有变化不需 migration script(老数据字段留着,新代码不读)。

### `users` 加 1 字段

```js
{
  // 现有(保留): _id, _openid, unionid?, wechat, createdAt, updatedAt, lastLoginAt
  groupId?: string,            // 用户群号,首次 RegisterProfile 时填,~30 字符
  registeredAt?: Date,         // groupId 设置之后 = now

  // 历史 OTP 字段(代码不再读写):name?, phone?
}
```

### `orders` 字段重构 + 加 2 字段

```js
{
  // 现有保留: _id, _openid, orderNo, outTradeNo, items[], amount, status, payStatus,
  //          shipping{}, shippingFee?, createdAt, updatedAt 等

  // ✏️ 改:remark → notes 子文档
  notes: {
    buyer?: string,    // 用户下单时填(取代既有的 order.remark)
    seller?: string,   // admin 后台填
  },
  // 旧的 order.remark 字段:新代码不写,只在 fallback 读
  //   显示逻辑:order.notes?.buyer ?? order.remark

  // 🆕 物流跟踪
  tracking?: {
    weight?: number,         // kg,e.g. 2.5
    courierName?: string,    // enum, e.g. "顺丰"
    courierNo?: string,      // 快递单号
    setAt?: Date,
  },

  // ✏️ userSnapshot 重定义(OAuth 信息 + 群号)
  userSnapshot: {
    nickname: string,       // wechat.nickname || '微信用户'
    avatar: string | null,  // wechat.avatar
    groupId: string | null, // user.groupId
    // 旧字段 name/phone 不再写
  },
}
```

### `products` 加 7 字段

```js
{
  // 现有保留: _id, title, desc?, coverFileId, category 等

  brand?: string,
  spec?: string,                  // "500g/盒"
  basePrice?: number,             // 分,正常收录价(reference)
  englishName?: string,
  courierName?: string,           // enum 见下
  courierFactor?: number,         // 0-10,按重量算运费的乘数
  secondaryImages?: Array<{ url: string, caption: string }>,
}
```

### 快递名 enum

```js
['顺丰', '中通', '圆通', '极兔', 'EMS', 'Australia Post', 'StarTrack', '其他']
```

实施时可微调,留个常量在 `cloudfunctions/_lib/constants.js`(若没有则建)。

## Section 2: Admin UI(web-admin)

### `web-admin/src/pages/OrderDetail.tsx`

新增 3 个 Card,放在现有 "运费尾款" Card 之后:

   - Textarea(max 500 字)
   - 保存按钮调新 cf `_admin/updateOrderNotes({orderId, sellerNote})`

   - 客户昵称(`order.userSnapshot.nickname`)显示在客户姓名行(replace 旧的 `userSnapshot.name`)

### `web-admin/src/pages/ProductEdit.tsx`

现有 title / 分类 / 主图 表单下方加 7 字段输入区(分组成 "商品资料" 块):

| 字段 | 控件 |
|---|---|

### `web-admin/src/pages/TuanEdit.tsx` 自动联动

| tuan_item 字段 | ← product 字段 |

| `title` | `product.title` |
| `price` | `product.basePrice`(用户可调 +- 任意) |
| 新展示行 | `product.spec / brand / englishName`(readonly) |

不破坏现有 copyFromTuan(整团复制)。只在新建 tuan_item 时联动。

### 部署流程

```
1. subagent-driven 跑 plan
2. 本地 node local-backend/api/test-shim.js → all pass
3. git push → CI test → Deploy App workflow → 自动 staging
4. 一次性 mongo 清:

     db.getSiblingDB(\"mogu_express\").otp_codes.drop()
   "'
5. 微信端到端验:
   - 新隐身/新设备 OAuth → /register-profile 看到群号 input
   - 填群号 → 进首页

   - admin 设运费 / 物流 / 卖家备注 → 用户端订单详情看到
```

### Verification(端到端必过)

- [ ] OAuth 进入 → 群号未填 → 跳 /register-profile,无 name/phone fields
- [ ] 填群号 → user.groupId 写入,registeredAt set → 跳回首页
- [ ] 下单 → 不报 "请先完善姓名电话" → 订单 userSnapshot = {nickname, avatar, groupId}
- [ ] admin OrderDetail 看到客户昵称 + 群号
- [ ] admin 设运费 / 物流 / 卖家备注 → 用户端订单详情看到对应信息
- [ ] admin 编辑商品 → 7 新字段保存,刷新仍在
- [ ] admin 创建 tuan_item 选 product → title/price/spec 自动 fill
- [ ] 浏览器(非微信)打开 → bootstrap 跳 /qr-fallback,不再去 /login

### Rollback

- 各 commit 独立成包,可单独 revert
- 删 OTP cf 的 commit 独立(便于回退取代码)
- OTP collection drop 不可逆,但 OTP 数据本身是临时性的(每 OTP 5 分钟过期),丢了无业务影响
- 若 createOrder 改 userSnapshot 形态出问题:旧订单 userSnapshot 字段保留(读时 fallback),不破坏历史
