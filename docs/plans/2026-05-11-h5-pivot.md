## H5 商城重做 — 替代微信小程序(2026-05-11)

### Context

甲方确认商品涉及保健品/营养补充剂/母婴/奶粉/零食/衣服等多品类,微信小程序"海外主体+跨境保健品"政策红线无法绕过。**决定转向独立 H5 商城路线**:客户在微信内打开 H5 链接下单,绕开小程序审核,合规零风险。

接龙团购模型保留(不改为常规商城)。鉴权用手机号 OTP,支付继续 HuePay 网页通道,部署到腾讯云轻量 HK + Cloudflare DNS。现有 `miniprogram/` 代码归档保留,业务逻辑层 `cloudfunctions/` 跟 `web-admin/` 95% 复用。

### 决策摘要

| 维度 | 选择 |
|---|---|
| 团购模型 | **接龙保留**(不改常规商城) |

| 技术栈 | Vite + React 18 + TypeScript + antd-mobile + Tailwind + Zustand + React Router |
| 工程结构 | Monorepo:`web-shop/` 新建,跟 `web-admin/` 同级 |

### 系统架构

```
客户(国内,微信内)              团长(澳洲,浏览器)
   ↓                              ↓
shop.xxx.com                   admin.xxx.com
(web-shop, 本期重点)            (web-admin, 不动)
   ↓ HTTPS                        ↓ HTTPS
        ↓
腾讯云轻量 HK(2C4G,¥24/月)
├─ Caddy(80/443,自动 LE 证书)
│   ├─ shop.xxx.com  → web-shop 静态
│   ├─ admin.xxx.com → web-admin 静态
│   └─ api.xxx.com   → api:4000
├─ api(Express + cloudfunctions shim)
├─ mongo(副本集 rs0)
└─ minio(开发) / 腾讯云 COS 香港(生产)

外联:HuePay 网页支付 API、腾讯云 SMS(OTP)
```

### 鉴权 + 支付流程(关键)

**A. 浏览阶段(游客免登)**:
- 所有商品浏览路由(`/`、`/tuan/:id`、`/product/:id`、`/cart`)全部开放
- 购物车未登录写 localStorage,登录后同步合并到服务端

**B. 下单触发登录**:
```
客户在 /cart 点「去结算」
  → 检测 token
    ├─ 已登录 → 跳 /checkout
    └─ 未登录 → 跳 /login?returnTo=/checkout
  → /login 页:输手机号 → sendOtp(腾讯云 SMS)
    → 输 6 位 OTP → verifyOtp → 签 JWT + 返 isRegistered
  → 首次注册(isRegistered=false)→ /register-profile(姓名+地址)
    → mergeCart(把 localStorage 上传合并)
  → 跳回 returnTo
```

**C. 受保护路由(需登录)**:
`/orders`、`/orders/:id`、`/profile`、`/addresses`、`/checkout`
React Router 用 `<ProtectedRoute>` HOC,未登录跳 `/login?returnTo=<original>`。

**D. 支付流程**:
```
/checkout 提交 → POST /api/createOrder
  → 后端事务:库存校验 + inc sold + 建 pending_pay 订单 + 调 HuePay createOrder
  → 返回 { orderId, payParams: { hueRedirectUrl } }
前端 window.location.href = hueRedirectUrl
  → HuePay 页面用户用微信/支付宝扫码付
  → HuePay 异步回调 /api/payCallback → 验签 → 更新 paid
  → HuePay 同步跳回 /pay-result/:orderId → 前端轮询 getOrderDetail
```

### 风险

1. **HuePay 网页通道 API** 跟小程序 JSAPI 不同,需找 HuePay 商务重新拿文档(M2' 阻塞项)
2. **腾讯云 SMS** 国内号短信签名审核需 1-3 个工作日,提早申请
3. **衣服 SKU 变体**:MVP 不做,团长每尺码独立录入;二期补 variant 模型
4. **微信内 H5 拉起 HuePay 跳转**:可能被微信拦截(微信对外部链接管控),需 HuePay 提供在微信浏览器内的兼容方案,或退化到"复制链接到浏览器打开"提示
5. **OTP 防刷**:必须做手机号频次限制(60s/次,每号每日 ≤10 次),后端 Redis 或 MongoDB TTL 索引实现
