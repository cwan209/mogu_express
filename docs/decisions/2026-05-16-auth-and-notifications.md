# 决策:登录 + 通知架构

> 2026-05-16 跟自己 + 甲方对齐(待甲方书面确认)。

## TL;DR

- **登录** → 微信 OAuth 静默,不发短信
- **通知** → 商城内小红点 / 弹窗 + 团长群里手动 @,**不发模板消息**
- **公众号** → 海外认证服务号(澳洲主体直接申请,USD 99/年,~2 周认证)

不做的:
- ❌ 国内腾讯云 SMS(避大陆主体 + ICP 备案)
- ❌ 公众号模板消息(避审核 + 等待)
- ❌ 国际通道 SMS 发大陆号(通过率差,UX 灾难)

## 为什么

### 登录走 OAuth 而不是 OTP
- 大陆手机号 SMS 是死结(国际通道通过率 70-85%,国内通道要 ICP + 大陆主体)
- 中国用户 99% 有微信
- 公众号 OAuth = 100% 通过率 + $0 + 用户无感
- 留 OTP 路径作 dev mode / 极少数非微信 fallback(显示二维码引导回微信)

### 通知走"小红点"而不是模板消息
- 模板消息要等公众号认证(2 周)+ 模板审核(1-3 天)
- 接龙团购本质是人情生意 → **团长在群里 @ 比系统自动推更亲切**
- 商城弹窗 + 红点足够覆盖回访用户(团购用户天然活跃)
- 数据看一段后真有漏付再上模板消息(增量,不冲突)

### 公众号走海外认证服务号
- 甲方澳洲公司直接能申请(无需大陆主体)
- 服务号才有 OAuth + JSSDK + 自定义菜单全套功能
- 海外服务号内容审核宽松(避当初保健品类问题)
- USD 99/年比代办大陆主体便宜

## 用户体验

### 登录(无感)
```
用户在微信里 → 公众号底部菜单"商城" → 点击
→ 微信内浏览器灰屏一闪(1-2 秒 OAuth)
→ 进首页,已登录,直接浏览/加购物车
```

### 首次下单
```
加购 → 结算 → 弹"填收货信息"(姓名/电话/地址)
→ 一次填完,以后自动带
→ 提交订单 → HuePay 支付 → 完成
```

### 团长结算运费 → 用户付尾款
```
团长:
   admin 后台选订单 → 输运费 ¥35 → 保存
   微信群里 @ 大家:"运费出来啦,看自己的订单"

用户:
   进商城 → 首页弹"您有 1 个待付订单 [前往支付]"
   或:"我的订单" tab 红点 → 进去看待付 ¥35
   点击 → HuePay 支付 → 完成
```

### 非微信打开(异常,<5%)
```
看到大二维码"请用微信打开"
长按识别 → 跳到公众号 → 走正常流程
```

## 未来可加(都不阻塞)

- **模板消息**:公众号跑稳后想加自动推就加,不冲突
- **国际版 SMS**:海外华人用户多+被非微信入口骂时再开
- **JSSDK 自定义分享卡片**:看分享数据决定 ROI(C 方案)

## 风险

| # | 风险 | 缓解 |
|---|---|---|
| 1 | 海外公众号高级接口"部分受限"(微信文档模糊) | 认证下来实测;如果 OAuth 真不可用,退到留 OTP + 引导扫码 |
| 2 | 微信群广播覆盖率不够(用户没看群消息漏付) | 商城内弹窗 + 红点是主要覆盖手段;数据观察后看是否补模板消息 |
| 3 | 极少数大陆号 + 不用微信的用户被排除 | UX 提示扫码;数据上看占比,真有非微信用户骂再考虑 |
